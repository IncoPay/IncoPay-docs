IncoPay is **open source** — no product tiers. This page covers **how you deploy** the pieces.

## Hosted facilitator (recommended)

A reference facilitator runs on Solana devnet — no infrastructure required:

```
https://inco-facilitator-production.up.railway.app
```

Point the SDK at this URL and you're done. The facilitator pays SOL fees on every settle from its own funded keypair, so end users are gasless. See [Hosted stack](/reference/hosted-stack) for the full config table.

**Best for:** demos, hackathons, AI-pay-per-call apps, anything where you don't want to run a paymaster.

## Self-hosted facilitator

Run [`IncoPay/inco-facilitator`](https://github.com/IncoPay/inco-facilitator) yourself when you need:

- A funded keypair under your control
- A different IncoToken mint / merchant pubkey
- Mainnet (when supported)
- Custom rate-limiting, logging, or cap policies

```bash
git clone https://github.com/IncoPay/inco-facilitator
cd inco-facilitator && npm install && npm run build
FACILITATOR_KEYPAIR='[1,2,...]' \
TOKEN_MINT=7crFMbJN7hxVhUPNcRRxTGr9nD3TnvpZ8pNZepA19wuB \
SOLANA_RPC_URL=https://api.devnet.solana.com \
npm start
```

Required env:

| Variable | Purpose |
|----------|---------|
| `FACILITATOR_KEYPAIR` | Inline JSON array `[1,2,...]` or base64 of the 64-byte secret. This pubkey signs as fee payer and as the `spender` on the on-chain `approve` allowance. |
| `TOKEN_MINT` | IncoToken mint pubkey on the target network |
| `SOLANA_RPC_URL` | RPC URL — defaults to `https://api.devnet.solana.com` |
| `PORT` / `FACILITATOR_PORT` | Listening port — defaults to `4021` (Railway/Render/Fly auto-set `PORT`) |

The facilitator stores session state in a local sqlite file (`DB_PATH`, default `./sessions.db`). On a multi-instance deploy, mount this on a shared volume or front it with a serializing proxy.

**Best for:** production deploys, custom merchants, anyone who can't trust a third-party paymaster.

## Frontend deploy

The IncoPay demo Next.js app at [`ayushsingh82/IncoPay`](https://github.com/ayushsingh82/IncoPay) deploys cleanly to **Vercel** — only the `solana-x402-sessions` SDK and a wallet adapter are needed on the client. Point `NEXT_PUBLIC_FACILITATOR_URL` at either the hosted facilitator or your own deploy and you're done.

## Kora (legacy)

Earlier versions of IncoPay used **Kora** as a separate paymaster service. The current architecture folds fee-payer responsibility into the facilitator itself, so **Kora is no longer required**. The `/pay/*` single-payment legacy endpoints can still proxy to Kora if `KORA_RPC_URL` is set, but the session scheme (the recommended path) does not need it.
