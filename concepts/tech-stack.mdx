## Client SDK

[`solana-x402-sessions`](https://www.npmjs.com/package/solana-x402-sessions) on npm.

- **`@solana/web3.js`** — `Transaction`, `Connection`, `PublicKey`
- **`@coral-xyz/anchor`** — IDL-driven IncoToken instruction encoders (peer)
- **`@inco/solana-sdk`** — `encryptValue` for the encrypted cap; `attested-decrypt` helpers if you decrypt balances client-side (peer; ESM, dynamically imported)
- **`solana-x402-sessions/scheme`** — Optional `SessionIncoScheme` plugin for `@x402/core` resource-server framework

## Frontend (demo app)

The IncoPay demo at [`ayushsingh82/IncoPay`](https://github.com/ayushsingh82/IncoPay):

- **Next.js 16** (React 19)
- **Solana Wallet Adapter** (`@solana/wallet-adapter-*`) — Phantom, Solflare, any Standard Wallet
- **`solana-x402-sessions`** — session creation + `wrapFetch`
- **`@x402/core` / `@x402/svm`** — optional, only if using the framework path

## Facilitator

[`IncoPay/inco-facilitator`](https://github.com/IncoPay/inco-facilitator) — Express server.

- **Express** — HTTP API (`/sessions`, `/verify`, `/settle`, `/supported`, `/health`, `/pay/*`)
- **`better-sqlite3`** — session state (id, user, spender, asset, recipient, cap, spent, expirationUnix, …)
- **`@coral-xyz/anchor` + `@solana/web3.js`** — building IncoToken instructions and submitting txs
- **`@inco/solana-sdk`** — `encryptValue` server-side for per-call amount ciphertexts
- **`tweetnacl`** — Ed25519 verify of the auth message
- **`uuid`** — session ids

The facilitator's own keypair (`FACILITATOR_KEYPAIR`) signs as fee payer on every transaction it submits — no separate paymaster service required.

## On-chain (devnet)

- **IncoToken** — Anchor program (confidential SPL with `approve` + `transfer_with_authorization`)
- **Inco Lightning** — External program + `inco-lightning` Rust crate (CPI from IncoToken). Covalidator (Inco's TEE worker) performs the actual FHE arithmetic.
